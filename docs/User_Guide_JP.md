## 1. ダッシュボードの見方

### 自動更新について

- **更新タイミング**: 毎朝6時に自動更新
- **更新内容**:
  - 前日の新規動画データを取得
  - 過去7日分の動画の指標を更新（YouTubeの指標は48-72時間で確定するため）
  - 月次KPIを再計算

### Page 1: 月次サマリーの活用

#### 見るべきポイント

1. **投稿本数**
   - 前月比で増えているか？
   - 目標投稿数（例: 月20本）を達成しているか？

2. **総視聴回数**
   - 前月比で伸びているか？
   - 投稿本数が増えても視聴回数が減っていないか？（質の低下）

3. **平均維持率**
   - 50%以上が理想（Shortsの場合）
   - 前月比で下がっている場合は内容見直しが必要

4. **日次推移グラフ**
   - ベースライン（過去動画の再生）が上がっているか？
   - 急激なスパイクがある日 = バズった動画を特定

#### アクションの例

- 視聴回数が減少 → Page 2で原因分析（維持率？投稿時間？尺？）
- 登録者が増えていない → Page 3でサムネイル・タイトルを見直し

### Page 2: 動画パフォーマンス分析の活用

#### バブルチャート（質 vs 量）の読み方

**4つのゾーン**:

1. **右上（Winner）**: 高再生 × 高維持率
   - これが「勝ちパターン」
   - 同じテーマ・構成で横展開すべき動画

2. **左上（Hidden Gem）**: 低再生 × 高維持率
   - 内容は良いがサムネ/タイトルが弱い
   - リメイクまたはサムネ変更の候補

3. **右下（Clickbait）**: 高再生 × 低維持率
   - サムネは強いが内容が期待外れ
   - 視聴者離れのリスクあり

4. **左下（Dead）**: 低再生 × 低維持率
   - 失敗動画
   - なぜ失敗したかを分析して次に活かす

#### 尺別分析テーブルの読み方

- 平均再生数が最も高い尺を確認
- 例: 「15-30秒」が最も伸びる場合、今後はその尺を中心に

#### 投稿時間ヒートマップの読み方

- 色が濃い = その時間帯の投稿が伸びやすい
- 最適な投稿時間を見つける
- 例: 「火曜21時」が最も伸びる → 今後はその時間に投稿

### Page 3: クリエイティブ分析の活用

#### タグ別パフォーマンス

- どの企画ジャンルが伸びているかを確認
- 例: 「ダンス」が伸びる → ダンス動画を増やす
- 「Untagged」が多い → VIDEO_TAGSでタグ付け必須

#### サムネイル・タイトル分析

- 視覚的にパターンを見つける
- 伸びているサムネイルの共通点:
  - 文字の大きさ
  - 色使い
  - 構図（顔のアップ？全身？）
- 伸びているタイトルの共通点:
  - 疑問形？断言形？
  - 数字の有無（「3つの理由」など）
  - 文字数

## 2. 動画へのタグ付け

### なぜタグ付けが必要か

- YouTube APIでは動画の「内容」は取得できない
- 手動でタグを付けることで、企画ジャンル別の分析が可能に

### タグ付けの手順

1. **VIDEO_TAGSシートを開く**
   - スプレッドシート「Shorts Analytics Data」を開く
   - 「VIDEO_TAGS」シートをクリック

2. **動画IDを確認**
   - A列に動画IDが自動入力されている
   - （もし空の場合は、GASで `populateVideoTags()` を実行）

3. **タグを入力**
   - B列（tag_category）にタグを入力
   - 例:
     - 「ダンス」
     - 「解説」
     - 「裏話」
     - 「チュートリアル」
     - 「Vlog」

4. **メモを入力（任意）**
   - C列（notes）に気付きや改善点を記録
   - 例:
     - 「BGMを大きくした」
     - 「テロップを減らした」
     - 「初めて顔出しした」

### タグ付けのベストプラクティス

- **一貫性を持つ**: 「ダンス」と「踊り」を混在させない
- **粒度を統一**: 細かすぎず粗すぎず
  - 悪い例: 「ダンス_ヒップホップ_初心者向け」（細かすぎる）
  - 良い例: 「ダンス」「解説」（適度）
- **5-10個のタグに収める**: 多すぎると分析が難しい

## 3. データ分析のポイント

### 週次レビュー（毎週月曜日）

1. **Page 1で先週の成果を確認**
   - 先週の投稿本数・総再生数
   - 日次推移でバズった動画を特定

2. **Page 2でバズった動画を深掘り**
   - バブルチャートで右上の動画を特定
   - その動画の尺・投稿時間・維持率を確認

3. **Page 3でサムネイル・タグを確認**
   - バズった動画のサムネイルパターン
   - どのタグが伸びているか

4. **次週の戦略を立てる**
   - 勝ちパターンを再現する企画
   - 投稿時間の最適化

### 月次レビュー（毎月1日）

1. **Page 1で月次KPIを確認**
   - 前月比で各指標が改善しているか
   - 目標達成度（例: 月間100万再生）

2. **失敗動画の分析**
   - Page 2のバブルチャート左下の動画
   - なぜ失敗したかを仮説立て

3. **タグ別の振り返り**
   - Page 3でどのジャンルが伸びたか
   - 来月の企画比率を調整

4. **PDFレポート作成**
   - Looker StudioでPDF出力
   - クライアントやチームメンバーに共有

## 4. 手動更新の方法

### 通常の手動更新

自動更新を待たずに今すぐデータを更新したい場合:

1. **スプレッドシートを開く**
   - 「Shorts Analytics Data」を開く

2. **Apps Scriptエディタを開く**
   - 「拡張機能」→「Apps Script」

3. **manualRefresh()を実行**
   - 関数選択ドロップダウンで「manualRefresh」を選択
   - 「実行」ボタンをクリック

4. **実行ログで進捗確認**
   - 下部の「実行ログ」タブで進捗を確認
   - 完了したら「✅ 日次更新完了」と表示される

### 全データ再取得（フルリフレッシュ）

データに不整合がある場合のみ使用:

1. **Apps Scriptエディタを開く**

2. **fullRefresh()を実行**
   - 関数選択で「fullRefresh」を選択
   - 「実行」ボタンをクリック

3. **確認ダイアログ**
   - 「既存データを全て削除して再取得します。よろしいですか？」
   - 「はい」をクリック

**注意**:
- APIクォータを大量に消費します
- 通常は使用しないでください

## 5. よくある質問（FAQ）

### Q1. 昨日投稿した動画がダッシュボードに表示されない

**A**: 自動更新は毎朝6時です。6時前に投稿した動画は翌朝6時に反映されます。すぐに確認したい場合は `manualRefresh()` を実行してください。

### Q2. 再生数がYouTube Studioと一致しない

**A**: 以下の理由が考えられます:
- YouTube Analyticsの指標は48-72時間で確定します
- Shortsアナリティックスは毎日、過去7日分を再取得して最新化しています
- 数日待てば一致します

### Q3. 収益データが「0」になっている

**A**: 収益指標はYouTubeパートナープログラム（収益化）に参加しているチャンネルのみ取得できます。収益化していない場合は常に0になります。

### Q4. CTR（クリック率）やインプレッションが「0」になっている

**A**: これらの指標も収益化が必要です。収益化していない場合は取得できません。

### Q5. VIDEO_TAGSでタグを付けたのに、Page 3に反映されない

**A**: VIDEO_TAGSでタグ付け後、加工シートの更新が必要です:
1. Apps Scriptエディタを開く
2. `refreshProcessedSheets()` を実行
3. VIDEO_MASTERシートのtag_categoryカラムが更新される
4. Looker Studioを開いて更新

### Q6. 自動更新が止まっている

**A**: トリガーを確認してください:
1. Apps Scriptエディタを開く
2. 左側メニュー「トリガー」（時計アイコン）をクリック
3. 「scheduledDailyUpdate」トリガーが存在するか確認
4. ない場合は `setupDailyTrigger()` を実行

### Q7. エラーメールが届いた

**A**: LOGシートでエラー内容を確認:
1. スプレッドシートの「LOG」シートを開く（非表示の場合は右クリック→「シートを表示」）
2. 最新のエラーメッセージを確認
3. エラー内容に応じて対処（[トラブル対処法](#7-トラブル対処法)参照）

## 6. メンテナンス

### 週次メンテナンス（自動）

- 毎週日曜日午前3時に自動実行
- LOGシートの古いエラーログ（30日以前）を削除
- 設定不要

### 月次メンテナンス（手動）

#### VIDEO_TAGSの整理

- 新規動画にタグを付ける
- タグの一貫性をチェック

#### データ量の確認

- RAW_VIDEO_DATAが2000行を超えたら、古いデータをアーカイブ
- 通常は数年分のデータでも問題なし

### API認証の更新

- OAuth認証は通常6か月有効
- 「YouTube APIの認証が必要です」エラーが出た場合:
  1. `resetAuth()` を実行
  2. `showAuthUrl()` で再認証

## 7. トラブル対処法

### 問題: 自動更新が失敗している

**確認方法**: LOGシートを開いて最新のエラーを確認

#### エラー: 「API quota exceeded」

**原因**: APIクォータ制限（10,000ユニット/日）に達した

**対処法**:
- 翌日まで待つ
- 動画数が多い場合は、取得頻度を週1回に変更:
  1. Triggers.gsの `setupDailyTrigger()` を編集
  2. `.everyDays(1)` を `.everyDays(7)` に変更

#### エラー: 「Service invoked too many times」

**原因**: 実行時間が6分を超えた

**対処法**:
- 動画数を減らす（古い動画データを削除）
- または取得範囲を短くする（Config.gsの `DAILY_LOOKBACK_DAYS: 7` を `3` に変更）

#### エラー: 「Authentication required」

**原因**: OAuth認証が失効した

**対処法**:
1. `resetAuth()` を実行
2. `showAuthUrl()` で再認証
3. 認証完了後、`manualRefresh()` でデータ取得

### 問題: Looker Studioにデータが反映されない

**対処法**:
1. Looker Studioで「更新」ボタンをクリック
2. それでも反映されない場合、データソースを再接続:
   - 「リソース」→「追加したデータソースを管理」
   - 各データソース →「編集」→「再接続」

### 問題: スプレッドシートが重い

**原因**: データ量が多い、または数式が多い

**対処法**:
1. 古いデータ（1年以上前）を別シートにアーカイブ
2. ブラウザのキャッシュをクリア
3. 別のブラウザで開く

## まとめ

### 日常的なワークフロー

1. **毎朝**: 自動更新（6時）
2. **毎週月曜**: ダッシュボードで週次レビュー
3. **毎月1日**: 月次KPI確認 → PDF出力 → 戦略見直し
4. **随時**: 新規動画にVIDEO_TAGSでタグ付け

### 成功のポイント

- データを見るだけでなく、**アクションに繋げる**
- 勝ちパターンを見つけたら**徹底的に横展開**
- 失敗動画も**学びの材料**として分析
- PDCAサイクルを回す（VIDEO_TAGSのnotesに記録）
